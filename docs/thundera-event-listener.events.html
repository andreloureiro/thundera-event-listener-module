thundera-event-listener.events

validate-incoming-event

Implements multimethods for each event type that this module will send to the Events API.

When a new type of event is created, a new validate-incoming-event must be created, otherwise there’ll be no routing to the schema validation.

Must receive an event type symbol and data object representing the payload data. If no key is found, raises an error.
